Deliverables: Sprint 0

User Story:
  • As a supervisor, Johan would like to log-in to the webapp and have it recognize his status as an administrator.
  • As a supervisor, Juan would like to log-in to the webapp and have it recognize his status as a client.
  • As a booker, Jennifer would like to log-in to the webapp and have it recognize her status as a booker.
Justification: Decided to tackle three user stories since they are integrated into a single log-in system, and it gives us a chance to experiment with PHP, CakePHP framework, and the MySQL Database System. Information for tech used.

Estimation Size: 13

Time Taken: Learning & Research (3 hrs 50 mins) + Time to Implement (10 mins) = 4 hrs

Manual Testing Instructions:
  • Use supervisor credentials to log-in (Login: admin/ Password: adminpass), login should be successful.
  • Use booker credentials to log-in (Login: booker/ Password: bookerpass), login should be successful.
  • Use client credentials to log-in (Login: user/ Password: userpass), login should be successful.
  • Check if "http://.../admin.php" can be accessed when not logged in. Page should be able to be accessed.
  • User incorrect credentials to get "wrong username/ password combination" message.
    • Subcase 1 : incorrect username with incorrect password
    • Subcase 2 : correct username with incorrect password
