User Stories

Here you will find User Stories for the Storefront Project.

Jennifer Wong

  • As a booker , Jennifer would like to be able to request meeting spaces, so that she doesn't have to e-mail the supervisors.
  • As a booker, Jennifer would like to be able to see available meeting spots, so that she has access to the information.
  • As a booker, Jennifer would like to be able to see the schedule by clients, so that she can relay the information to the client.
  • As a booker, Jennifer would like to be able to see the upcoming daily schedule, so that she can prepare for the day.
  • As a booker, Jennifer would like to be able to see the upcoming schedule in a weekly form, so that she can prepare for the week.
  • As a booker, Jennifer would like to see the upcoming schedule in a monthly form, so that she can prepare for the month.
  • As a booker, Jennifer would like to be able to see information on missed booking from a client, so she can make informed decisions.
  • As a booker, Jennifer would like to be able to put clients on a waiting list for a particular meeting space, so that she keep track of the information.
  • As a booker, Jennifer would like to be kept updated on changes to the schedule, so that she can keep organized.
  • As a booker, Jennifer would like to be able to see a history of changes, so that she can keep organized.
  • As a booker, Jennifer would like to be able to undo an action, in case she makes a mistake.
  • As a booker, Jennifer would like to be able to input client information, so that she can keep track of the clients.
  • As a booker, Jennifer would like to be able to access this information on-the-go, so that she can be on top of the Storefront schedule.
  • As a booker, Jennifer would like to be able to input room information, so that she can relay the information to the client.

Juan Martinez

  • Juan would like to view the contact details of his students so that he can contact them in the event of an emergency (i.e. class is cancelled) and not let them down.
  • As a student of Juan, I would like to receive a notification by telephone or email if a class is cancelled.
  • Juan would like to view how many students are going to attend his art class so that he can bring the necessary amount of materials.
  • Juan would like to view which rooms are available on a specific date so that he can schedule his art classes accordingly.
  • Juan would like to book a room at a specific date and time for his art class so that he can conveniently book rooms without going to Storefront, as it is not always convenient for him.
  • As a student of Juan, I would like to sign up his for his art classes online so that it is much more convenient than calling in.
  • As a student of Juan, I would like to view the details of a class that I will be signing up for so that I can see whether I will be interested in the lesson.
  • Juan would like to sign up for a cooking class with the online system so that he can secure his spot in the class.
  • Juan would like to see the number of remaining spots for the cooking class so that he can see how many friends he can invite to the class.
  • Juan would like to see a calendar view of his classes so that he can see which classes he can have a better view of which classes he will be teaching, and which ones he will be attending.
  • Juan would like a tablet friendly interface so that he can access and book rooms using his iPad, as he does not always have access to his laptop.

Jahan Malik

  • As a supervisor, Jahan would like to be able to access and edit the schedule from any computer, so he can more efficiently book clients.

    • Test Cases:
    • Attempt to access and login to the website from multiple OS/web browsers (must check with supervisor to know which web browsers/OS will be used)
    • Check if login is successful

  • As a supervisor, Jahan would like to have access to a client database where he can add and remove clients, to keep track of them.

    • Test Cases:
    • Add a new client to an empty database, check that correct client is in the database
    • Remove that client, check that database is empty
    • Add a new client to non-empty database, check that correct clients are in database
    • Remove a client, check that correct clients remain in database

  • As a supervisor, Jahan would like to be able to view the schedule by day, week, month, or room, to prepare for the future.

    • Test Cases:
    • Book client for a timeslot, view schedule for that day and check if booking can be seen
    • View schedule by week, check if booking is on correct day
    • View schedule by month, check if booking is on correct day of correct week
    • View schedule by room, check if correct timeslot is booked

  • As a supervisor, Jahan would like to be able to view a clientâ€™s booking history, for data analysis.

    • Test Cases:
    • Book one or more timeslots for a single client
    • View that client's history and check that correct timeslots are present

  • As a supervisor, Jahan would like to be able to view room booking history, for data analysis.

    • Test Cases:
    • Book multiple clients to a single room for different times
    • View booking history and check that correct clients are present along with corresponding times

  • As a supervisor, Jahan would like to be able to schedule official booking times for clients, to keep track of information.

    • Test Cases:
    • Book a client at a specific time and room, check that booking schedule holds the correct information

  • As a supervisor, Jahan would like to be able to delete bookings from the schedule using a confirmation prompt, to free up timeslots and prevent accidental deletion.

    • Test Cases:
    • Book a timeslot for a client, then attempt to delete that booking
    • Cancel deletion using prompt, check that booking still remains in timeslot
    • Attempt to delete again, approve through the prompt, check that the timeslot is empty

  • As a supervisor, Jahan would like to have an undo delete option, in case of an accidental deletion.

    • Test Cases:
    • Book a timeslot for client, delete the booking (approve through prompt), check that the timeslot is empty
    • Undo the action, check that the timeslot is occupied by the correct client

  • As a supervisor, Jahan would like to be able to approve booking requests made by the counter staff, to prevent unnecessary emails and increase efficiency.

    • Test Cases:
    • Make a request for booking a client, check that supervisor can see request and that the corresponding timeslot is still empty
    • Approve the request, check that the timeslot is booked by the correct client

  • As a supervisor, Jahan would like to be able to add clients to a waitlist, for efficiency and organization.

    • Test Cases:
    • Book a client for a non-empty timeslot with no waitlist, check that original client is still booked
    • View corresponding waitlist, check that only the newly booked client is present
    • Book a client for a timeslot with a non-empty waitlist, check that original client is still booked
    • View corresponding waitlist, check that it contains correct clients

  • As a supervisor, Jahan would like the next client in a waitlist to be approved automatically if the previous client cancels, to save time and keep track of information.

    • Test Cases:
    • Delete a booking which has a waitlist with one client, check that the timeslot now contains the wait-listed client
    • Check that the waitlist is now empty
    • Delete a booking which has a waitlist with multiple clients, check that the timeslot now contains the first client from the waitlist
    • Check that the remaining clients are still in waitlist in correct order

  • As a supervisor, Jahan would like a coloured visual representation of different types of bookings, including request, normal, waitlist, and finished, for organization and ease of use.

    • Test Cases:
    • Request a booking, check that correct colour is assigned to timeslot
    • Make official booking, check that correct colour is assigned to timeslot
    • Create a waitlist, check that correct colour is assigned to timeslot
    • Make official booking, set status to finished, check that correct colour is assigned to timeslot

  • As a supervisor, Jahan would like to be able to book a client for multiple meeting spaces at once, to save time and prevent repetitive work.

    • Test Cases:
    • Book a client for a single room at 2 or more different timeslots, check that client is present in all timeslots
    • Book a client for 2 or more rooms at a single timeslot, check that all rooms contain correct client
    • Book a client for 2 or more rooms at 2 or more different timeslots, check that corresponding rooms and times contain client

  • As a supervisor, Jahan would like to be able to select any booking and view/edit all details related to it (must talk to supervisor to determine necessary details), for staying prepared and organized.

    • Test Cases:
    • Book a timeslot, see if correct details can be viewed
    • Modify a detail, see if properly changed (repeat for each detail)
